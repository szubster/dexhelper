# Git Workflow Rules

Always follow these rules for version control:

1. **Branch Management**: At the start of every session/task, create a new branch from `main` (or the default branch) with a descriptive name.
2. **Commit Checkpoints**: Make atomic commits at logical checkpoints throughout the development process (e.g., after completing a feature component, fixing a bug, or passing a set of tests).
3. **Session Completion**: Push the branch to the remote repository once the task is complete or at the end of the session.
