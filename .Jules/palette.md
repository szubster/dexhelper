## 2026-04-14 - Focus States and Labels on Custom Inputs
**Learning:** Custom styled buttons and file inputs without explicit labels and proper focus styling create significant accessibility barriers. Specifically, a missing `aria-label` on an icon-only button and a custom file upload wrapped in a `label` without accessible text `.sr-only` result in an ambiguous experience for screen readers. Furthermore, lack of `focus-visible` rings breaks keyboard navigation.
**Action:** Always verify that icon-only interactive elements contain `aria-label`. Ensure custom file inputs include `.sr-only` descriptive text within their wrappers. Use Tailwind `focus-visible:ring-2 focus-visible:ring-[var(--theme-primary)] focus-visible:ring-offset-2 focus-visible:ring-offset-zinc-950` utility classes to properly indicate focus.
