- **Coverage Tooling:** Use `pnpm test -- --coverage` combined with specific test runs `pnpm test src/path/to/test` to pinpoint coverage gaps.
- **IndexedDB Fallbacks:** `idb` has fallback mechanisms when it fails (like when the user rejects saving). It can be mocked via `vi.mock('idb', ...)` and you can forcefully reject `openDB` to trigger those try/catch blocks. Mocking internal module state for singletons requires `vi.resetModules()` and dynamic `import()` of the module within each test.
- **Strict Types:** Using `as unknown as Type` is essential for passing the strict linter rules that prevent `noExplicitAny`, for example, when mocking an external library like `idb`.
