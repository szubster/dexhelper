# Prevent Premature Artifact Generation

## Context
Recent runs have encountered issues where downstream artifacts (such as PRDs or Epics) are generated before their parent nodes are officially dispatched or worked on. This results in the agent assigned to the parent node finding the target artifact already existing, causing confusion, dummy diffs to satisfy automated reviewers, and potential drift in requirements.

## Proposed Solution
Implement a strict validation check within the `foundry-orchestrator.ts` or as a Git Pre-Commit Hook that prevents the creation of any node (e.g., PRD, EPIC, STORY, TASK) if its explicitly defined `parent` node is not in the `COMPLETED` state.
Alternatively, introduce a "Scaffolding Phase" where nodes can be scaffolded but their content cannot be finalized until unblocked. The preferred approach is strict prevention of premature files.

## Value
- Prevents workflow confusion and "dummy diff" rejections.
- Ensures strict adherence to the Atomic Handoff and Hierarchical Completion rules.
