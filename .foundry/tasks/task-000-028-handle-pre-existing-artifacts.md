# Handle Pre-existing Artifacts Rule

Added a new instruction to the `product_manager` persona regarding handling pre-existing target artifacts. When they find out that a target artifact such as a downstream PRD unexpectedly exists prior to the session, they should log this anomaly as a journal entry and create a new task/idea for the Agile Coach to review later.
